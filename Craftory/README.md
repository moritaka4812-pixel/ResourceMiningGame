# ResourceMiningGame（開発中）

MonoGame を用いて、**UI フレームワークとゲーム基盤を自作しているプロジェクト**です。  
ゲームとしてはまだ初期段階ですが、UI システムやレイアウト処理など、基盤部分の設計と実装を中心に進めています。
2026年6月開始

---

##  プロジェクト概要

このプロジェクトでは、MonoGame 上で動作する **独自 UI フレームワーク** を構築しています。  
Relative / Anchor を用いたレイアウトシステム、UIElement の責務分離、UIFactory による UI 生成の統一など、  
UI の基盤部分を重点的に設計しています。

ゲーム本体はこれから拡張していく予定ですが、  
UI とゲーム基盤の構造を整えることを優先しています。

---

##  主な実装要素

### ■ UIElement 基盤
- すべての UI の共通基底クラス  
- 位置・サイズ・描画・更新の基本処理を統一  
- 子要素を持つ構造を想定した設計

### ■ Button / TextLabel の責務分離
- Button：入力処理とクリック判定  
- TextLabel：テキスト描画に特化  
- UIElement の継承構造を活かした明確な責務分離

### ■ Relative / Anchor レイアウトシステム
- 親要素に対する相対位置指定（Relative）  
- 画面端基準の固定配置（Anchor）  
- 2 種類のレイアウトを組み合わせて柔軟な UI 配置を実現

### ■ SetUIElements（レイアウトパイプライン）
- UIElement の位置・サイズを一括で再計算  
- 画面サイズ変更や UI 更新に対応  
- UI のレイアウト処理を UI ロジックから分離

### ■ UIFactory（UI 生成の統一）
- Button / TextLabel などの UI を一元的に生成  
- フォントや GraphicsDevice の依存を集約  
- UI の追加・変更に強い構造
